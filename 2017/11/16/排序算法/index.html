冒泡排序（bubble-sort）

算法实现：
算法思路：

从数组中找到最小的元素放到a[0]，然后找到第二小的元素放到a[1]，以此类推。内层循环每次循环整个数组或者数组的一部分，通过两两比较，把最小的数冒泡到最前面。外层循环循环n-1次（0到n-2），第1次在a[0]得到最小的元素，第2次循环在a[1]得到第二小的元素….，第n-1次在a[n-2]得到倒数第二小的元素。

第一次外层循环，j的取值范围为（0，n-2），找到（0，n-1）上的最小值，放到a[0]
第二次外层循环，j的取值范围为（1，n-2），找到（1，n-1）上的最小值，放到a[1]
第二次外层循环，j的取值范围为（2，n-2），找到（2，n-1）上的最小值，放到a[2]
….

最后一次外层循环，j = n-2，此时找到a[n-2] 和 a[n-1]的最小值放到a[n-2]，这样a[0]到a[n-2]都是各自位置上的最小值了，所以a[n-1]自然是最大值。

因为是从小到大排序，所以内层应该从尾部开始交换，把最小值向左边冒泡，所以当i取最小值时，j应该有最大的取值范围（0，n-2），而且j的值应该递减，因为j是作为数组下标的，比如当j取最大范围的时候（第一次外层循环），应该按如下顺序进行两两交换：

a[n-1] <--> a[n-2]
a[n-2] <--> a[n-3]
...
a[1] <--> a[0] ==> 此时a[0]是整个数组的最小元素

所以整个循环执行的过程如下：

假设外层循环从 0到n-2

i = 0，    j 从 n - 2递减到0     ==> a[0]
i = 1，    j 从 n - 2递减到1     ==> a[0],a[1]
…
i = n-2，   j 从 n - 2递减到n-2    ==> a[0],a[1]...a[n-1]

算法分析：

最好时间复杂度：O(n)
当所给的数组全部已经排序好的时候，此时外层循环只有一次机会满足，内层循环只会循环一次 ,1 x (n - 1)。

最坏时间复杂度：O(n^2)



插入排序（insertion-sort）

算法实现：
算法思路：

把数组分为A和B两部分（假想的），A中的元素永远是排序好的，然后把B中的元素从左到右不断插入A中，插入之后，要把A中，从插入位置到结尾的元素整节往后挪一个位置（A整体也扩大了一个位置），等到B把所有元素都插到了A中，也就是A等于整个数组的长度时，数组也就排序好了。

所以这里外层循环应该循环B，内层循环用来将插入位置和元素原来位置之间的元素整体移动一个位置，A中的元素个数从1依次增大。

外层循环i代表B数组的元素下标，内层循环j代表数组A的元素下标。每一次外层循环中，将B中的元素从右到左，依次比较，两两交换，又因为A中的元素之前是有序的，每进行一次两两交换，都相当于把A从交换位置开始到结尾那段有序的向后挪了一个位置。

算法分析：

最好时间复杂度：O(n)
当所给的数组全部已经排序好的时候，内层循环每次都不需要循环，只需与A的最后一个元素比较一次即可退出 ,（n-1）x 1。

最坏时间复杂度：O(n^2)
当所给数组是反向排序的时候，内层循环每次都要从A的末尾比较并交换到A的开头。



二分插入排序（binary-insertion-sort）

算法实现：
算法思路：

二分插入在插入排序基础上做了优化，因为A是排序好了的，所以可以通过二分法找到B中的元素在A中的插入位置，而不是从A的末尾一直顺序比较交换。

二分查找最后一次进入while循环时，left = right = mid，此时只要比较此位置和insert的大小，比这个元素大，则插入位置在此位置右边，反之则此位置就是插入位置（这里如果相等也直接插在左边），因为最后会把插入位置连同后面的元素整体向后挪一个位置。这里用right + 1表示插入位置。

算法分析：

最好的时间复杂度：O(nlogn)
如上分析，二分插入只是减少了插入元素在A中比较的次数。当这个数组已经排序好了的时候，二分插入内层还需要经过一个while循环来确定插入的位置，而普通插入排序直接与A最后一个元素比较即可。

最坏的时间复杂度：O(n^2)
除了找到插入点所需的操作数从 O(n) 降为 O(logn)外，其它的操作并未减小，其时间复杂度依旧是 O(n^2)。例如当通过二分查找找到插入位置位于A的最前面，则内层循环还是要经过一整个循环进行元素移动操作。

如上分析，二分插入只是减少了插入元素在A中比较的次数。在最优情况可能会比普通插入略差，但总体回比普通插入稳定。



选择排序（selection-sort）

算法实现：
算法思路：

选择排序和冒泡排序的思路基本类似，为了方便理解这里同样把数组分为A和B两部分，A中的元素从一个开始，外层循环用来获得A中最后一个位置的值。比如第一次外层循环的时候，A只有一个元素即a[0]，B包括除了a[0]之外的所有元素，此时只要找到B中的最小元素与a[0]进行比较交换。第一次获得a[0]，第二次获得a[1]….

整体思路与冒泡基本一致，只是内层循环细节与冒泡不同。

内层循环的目的跟冒泡排序一样同样是确定数组子序列的最小值，只不过不是像冒泡排序那样两两比较，将最小值通过冒泡的方式传递到最前面，而是通过定义一个数组下标索引值min，刚开始将min定最小值为A的最后一个元素的索引，即令min = i，将B中的元素循环依次与a[min]比较，将两个值中的较小值的索引再赋给min，则内层循环过后，得到的min就是A最后一个元素和B这两个组成部分的最小值的索引，如果这个min改变了，说明B中存在比A最后的元素小的元素，将这两个值交换即可。

算法分析：

最好的时间复杂度：O(n^2)
当所给数组是完全排序好的时候，选择排序的内层循环依然需要执行，而上述的冒泡排序通过一个布尔值对执行过程进行了优化，内层只需要循环一次。

最坏的时间复杂度：O(n^2)
在最坏时间复杂度上与冒泡排序一样，但相对冒泡排序，在一定程度上减少了数据交换的操作。



归并排序

算法实现：
算法思路：

以上的排序方法都采用的是增量的思想，而归并排序采用的是一种分治的思想。算法的思路是：

设定一个合并过程，合并两个已经排序好的数组，假定为A和B，使合并后的数组也是排序好的。这个合并过程通过从小到大不断比较A和B的元素的大小，将较小的值放入一个新的数组中。此过程只要所给的A和B时有序的，最后就能得到一个有序的合并数组。

归并排序的过程就是将原数组不断分割，当分割到A和B都只有一个元素的时候，A和B肯定是排序好的，此时只要调用上面的合并过程，就可以得到一个包含2个元素的有序数组。由此，只要递归此过程，A和B中各自的元素总是保持有序的，当A+B等于数组的长度时，就能得到一个有序数组。
